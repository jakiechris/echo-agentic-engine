# Router Layer (路由层)

## 层级定位

Router Layer 是 Engine 对外暴露 HTTP 接口的路由控制层,负责接收 HTTP 请求、协调各业务模块完成请求处理、返回 HTTP 响应。它是 FastAPI 路由函数的集合,作为 HTTP 接口与业务模块之间的桥梁。

## 核心功能

1. **路由分发**: 接收 HTTP 请求,根据路径和方法分发到对应的处理函数
2. **参数提取**: 从 Request 中提取 Headers、Path Parameters、Body 参数
3. **模块协调**: 调用 API Layer 和 Core Layer 的模块完成业务逻辑
4. **响应构建**: 将业务模块的返回结果转换为 HTTP Response
5. **异常处理**: 捕获业务异常并转换为 HTTP 错误响应

## 触发的流程

Router Layer 触发 **9大流程中的前5个流程**(流程 3.2.1 - 3.2.5),这些流程均由 HTTP 请求触发:

| 流程编号 | 流程名称 | 触发路由模块 | HTTP接口 | 触发方式 |
|---------|---------|------------|---------|---------|
| **3.2.1** | OpenCode API 代理主流程 | ProxyRouter | `ANY /trans/{path}` | Echo-Agent 发送请求 |
| **3.2.2** | 管理接口：列出所有沙箱 | ListSandboxRouter | `POST /admin/sandbox/list` | 管理员查询沙箱列表 |
| **3.2.3** | 管理接口：查询单个沙箱 | GetSandboxRouter | `POST /admin/sandbox/get` | 管理员查询沙箱详情 |
| **3.2.4** | 管理接口：创建沙箱 | CreateSandboxRouter | `POST /admin/sandbox/create` | 管理员手动创建沙箱 |
| **3.2.5** | 管理接口：销毁沙箱 | DestroySandboxRouter | `POST /admin/sandbox/destroy` | 管理员手动销毁沙箱 |

**流程触发特点**:
- ✅ **实时触发**: 每次收到 HTTP 请求立即触发相应流程
- ✅ **并发处理**: 多个请求可并发触发多个流程实例
- ✅ **用户驱动**: 流程由外部用户(Echo-Agent 或管理员)主动调用触发

**未触发的流程**:
- ❌ 流程 3.2.6 - Engine 启动主流程 (由 Bootstrap Layer 触发)
- ❌ 流程 3.2.7 - HealthCheckTask 调用链 (由后台任务定时触发)
- ❌ 流程 3.2.8 - IdleCleanupTask 调用链 (由后台任务定时触发)
- ❌ 流程 3.2.9 - ConfigSyncTask 调用链 (由后台任务定时触发)

## 调用关系

- **被调用**: 由 FastAPI 框架自动调用
- **调用下游**:
  - API Layer: RequestValidator (参数验证), ResponseBuilder (响应构建)
  - Core Layer: SandboxManager (沙箱管理), RequestProxy (请求代理), ConfigManager (配置管理)
- **中间件**: 使用 FastAPI 中间件进行日志记录、异常处理

## 设计原则

- **薄控制层**: 路由函数仅负责协调模块调用,不包含业务逻辑
- **统一异常处理**: 所有异常由中间件统一捕获并转换为标准错误格式
- **异步优先**: 使用 async/await 提高并发性能
- **依赖注入**: 使用 FastAPI 的依赖注入机制管理模块实例

### 0.1 ProxyRouter 模块

**职责**: 处理 OpenCode API 代理请求,协调沙箱管理和请求代理

**HTTP接口**: `ANY /trans/{path}`

**参与流程**: 1 个流程 (3.2.1 - OpenCode API 代理主流程)

---

#### 0.1.1 handleProxyRequest

**功能描述**: 处理 OpenCode API 代理请求,自动处理沙箱创建、健康检查、故障恢复,并将请求代理到沙箱内的 OpenCode Serve

**流程**: 3.2.1 - OpenCode API 代理主流程

**HTTP定义**:
```python
@router.api_route("/trans/{path:path}", methods=["GET", "POST", "PUT", "DELETE", "PATCH"])
async def handleProxyRequest(
    path: str,
    request: Request,
    x_domain_id: str = Header(..., alias="X-Domain-ID"),
    x_sandbox_id: str = Header(..., alias="X-Sandbox-ID")
):
    """
    处理 OpenCode API 代理请求

    Args:
        path: OpenCode API 路径 (如 api/chat, api/files/read)
        request: FastAPI Request 对象
        x_domain_id: 租户标识 (从 X-Domain-ID Header 提取)
        x_sandbox_id: 沙箱标识 (从 X-Sandbox-ID Header 提取)

    Returns:
        Response: OpenCode 原生响应
    """
    pass
```

**入参**:

| 字段名 | 类型 | 来源 | 必填 | 说明 |
|--------|------|------|------|------|
| path | str | Path Parameter | 是 | OpenCode API 路径 (如 `api/chat`, `api/files/read`) |
| request | Request | FastAPI | 是 | FastAPI Request 对象,包含 Headers 和 Body |
| x_domain_id | str | Header (`X-Domain-ID`) | 是 | 租户标识,用于沙箱管理 |
| x_sandbox_id | str | Header (`X-Sandbox-ID`) | 是 | 沙箱标识,用于沙箱定位 |

**入参示例**:
```http
POST /trans/api/chat HTTP/1.1
Host: localhost:8000
X-Domain-ID: tenant-001
X-Sandbox-ID: sb-12345
Content-Type: application/json
Authorization: Bearer xxx

{
    "message": "Hello, OpenCode!"
}
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| Response | JSON | OpenCode 原生响应或标准错误响应 |

**出参示例**:

成功响应 (HTTP 200):
```json
{
    "status": "success",
    "data": {
        "response": "Hello! How can I help you?"
    }
}
```

失败响应 (HTTP 503):
```json
{
    "status": "error",
    "error": {
        "code": "SANDBOX_UNAVAILABLE",
        "message": "沙箱不可用",
        "details": {
            "domainID": "tenant-001",
            "sandboxID": "sb-12345"
        }
    }
}
```

**内部调用流程**:
1. **参数验证**: `RequestValidator.validateProxyRequest(request.headers)` → 提取并验证 domainID, sandboxID
2. **获取沙箱**: `SandboxManager.getSandbox(domainID, sandboxID)` → 不存在则自动创建,不健康则自动重建
3. **代理请求**: `RequestProxy.proxyRequest(sandbox, method, path, headers, body)` → 转发到沙箱内 OpenCode
4. **更新活跃时间**: `RedisClient.updateLastActive(domainID, sandboxID)` → 更新 lastActiveAt
5. **返回响应**: `ResponseBuilder.buildSuccessResponse(opencode_response)` → 返回 OpenCode 原生响应

**异常处理**:
- `ValidationError` → `ResponseBuilder.buildErrorResponse(400, ...)`
- `SandboxUnavailableError` → `ResponseBuilder.buildErrorResponse(503, ...)`
- `SandboxTimeoutError` → `ResponseBuilder.buildErrorResponse(503, ...)`
- `Exception` → `ResponseBuilder.buildErrorResponse(500, ...)`

## 中间件设计

### 异常处理中间件

**功能**: 统一捕获所有路由函数抛出的异常,并转换为标准错误响应格式

**实现方式**:
```python
@app.exception_handler(Exception)
async def global_exception_handler(request: Request, exc: Exception):
    return ResponseBuilder.handleException(exc)
```

### 日志中间件

**功能**: 记录所有请求的访问日志,包括请求路径、方法、参数、响应时间

**实现方式**:
```python
@app.middleware("http")
async def log_requests(request: Request, call_next):
    start_time = time.time()
    response = await call_next(request)
    process_time = time.time() - start_time
    logger.info(f"{request.method} {request.url.path} - {response.status_code} - {process_time:.3f}s")
    return response
```

```
│  └───────────────────────────────────────────┘  │
└─────────────────┬───────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────┐
│           API Layer (接口层)                     │

└─────────────────┬───────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────┐

                  │
                  ▼

```
