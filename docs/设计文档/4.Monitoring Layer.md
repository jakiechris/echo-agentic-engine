# Monitoring Layer (监控层)

## 层级定位

Monitoring Layer 负责监控沙箱运行状态,包括健康检查、资源监控、空闲监控和日志读取。

## 核心功能

1. **健康监控**: 检查沙箱健康状态,发送 HTTP 健康检查请求
2. **资源监控**: 读取沙箱进程的 CPU、内存占用
3. **空闲监控**: 扫描空闲超时沙箱,触发清理
4. **日志读取**: 读取沙箱 stdout/stderr 日志文件

## 调用关系

- **被调用**: 由 Core Layer 或 API Layer 调用
- **调用外部**:
  - HTTP 客户端: HealthChecker 发送健康检查请求
  - 系统命令: ResourceMonitor 读取进程资源占用
  - 文件系统: LogReader 读取日志文件
- **不调用上层**: 不依赖 Core Layer, API Layer

## 设计原则

- 只读操作: 仅读取状态,不修改沙箱状态
- 异步非阻塞: 健康检查、资源监控使用异步调用
- 超时控制: 健康检查设置超时 (默认 5 秒)

## 模块设计

### 4.1 HealthChecker 模块

**职责**: 沙箱健康检查,通过 HTTP 请求探测沙箱状态

**参与流程**: 5 个流程 (3.2.1 - 3.2.4, 3.2.7)

---

#### 4.1.1 checkHealth

**功能描述**: 检查单个沙箱的健康状态,通过发送 HTTP 健康检查请求探测沙箱是否正常运行

**流程**: 3.2.1 - OpenCode API 代理主流程<br>3.2.3 - 管理接口：查询单个沙箱<br>3.2.4 - 管理接口：创建沙箱

**入参**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| sandbox | Sandbox | 是 | 沙箱实例对象 |

**入参示例**:
```python
{
    "domainID": "tenant-001",
    "sandboxID": "sb-12345",
    "pid": 12345,
    "port": 30001,
    "nasPath": "/nas/tenant-001/sb-12345",
    "password": "uuid-password",
    "status": "running",
    "createdAt": "2026-03-14T10:30:00Z",
    "lastActiveAt": "2026-03-14T12:45:30Z"
}
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| healthStatus | str | 健康状态: `healthy` 或 `unhealthy` |

**出参示例**:
```python
# 健康状态
"healthy"

# 不健康状态
"unhealthy"
```

**内部处理**:
1. 构建 HTTP 请求: `GET http://127.0.0.1:{sandbox.port}/global/health`
2. 设置超时时间: 5 秒
3. 发送 HTTP 请求
4. 判断响应:
   - HTTP 200 且响应体 `{"healthy": true}` → 返回 `healthy`
   - HTTP 非 200 或超时或异常 → 返回 `unhealthy`

---

**入参**:

```python
        "pid": 12345,
        "port": 30001,
        "nasPath": "/nas/tenant-001/sb-12345",

        "status": "running",

        "lastActiveAt": "2026-03-14T12:45:30Z"

    {
        "domainID": "tenant-002",
        "sandboxID": "sb-67890",

        "status": "running",
        "createdAt": "2026-03-14T11:00:00Z",
        "lastActiveAt": "2026-03-14T13:00:00Z"
    }
]
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| healthMap | Dict | 健康状态映射,包含 `healthy` 和 `unhealthy` 两个列表 |


**内部处理**:
1. 使用并发机制(asyncio.gather 或线程池)对每个沙箱调用 checkHealth()
2. 收集所有沙箱的健康检查结果
3. 根据结果分类:
   - `healthy` → 加入 healthy 列表
   - `unhealthy` → 加入 unhealthy 列表
4. 返回 `{healthy: [...], unhealthy: [...]}`

### 4.2 ResourceMonitor 模块

**职责**: 资源监控,读取沙箱进程的 CPU、内存占用

**参与流程**: 2 个流程 (3.2.2, 3.2.3)

---

#### 4.2.1 getResourceUsage

**功能描述**: 读取沙箱进程的资源占用情况,包括 CPU 使用率和内存占用(RSS、VSZ)

**流程**: 3.2.2 - 管理接口：列出所有沙箱<br>3.2.3 - 管理接口：查询单个沙箱

**入参**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pid | int | 是 | 进程 ID |

**入参示例**:
```python
pid = 12345
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| ResourceUsage | Dict | 资源占用对象,包含 CPU 和内存信息 |

**出参示例**:
```python
{
    "cpu": 15.5,
    "memory": {
        "rss": 256,
        "vsz": 512,
        "unit": "MB"
    }
}
```

**ResourceUsage 对象字段说明**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| cpu | float | CPU 占用率 (百分比) |
| memory | Dict | 内存占用信息 |
| memory.rss | int | 实际物理内存占用 (MB) |
| memory.vsz | int | 虚拟内存占用 (MB) |
| memory.unit | str | 内存单位,固定为 "MB" |

**内部处理**:
1. 读取 `/proc/{pid}/stat` 文件
2. 解析进程状态信息:
   - 提取 RSS (Resident Set Size): 实际物理内存占用
   - 提取 VSZ (Virtual Memory Size): 虚拟内存占用
3. 计算 CPU 占用率:
   - 读取 `/proc/{pid}/stat` 中的 utime、stime 字段
   - 结合系统总 CPU 时间计算占用率
4. 转换单位:
   - RSS、VSZ 从页数转换为 MB (页大小通常为 4KB)
5. 返回 `{cpu: float, memory: {rss: int, vsz: int, unit: "MB"}}`

### 4.3 IdleMonitor 模块

**职责**: 空闲超时监控,识别长时间未活跃的沙箱

**参与流程**: 1 个流程 (3.2.8)

---

#### 4.3.1 scanIdleSandboxes

**功能描述**: 扫描所有沙箱,识别空闲超时的沙箱,用于触发清理操作

**流程**: 3.2.8 - IdleCleanupTask 调用链

**入参**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| sandboxes | List[Sandbox] | 是 | 沙箱实例列表 |

**入参示例**:
```python
[
    {
        "domainID": "tenant-001",
        "sandboxID": "sb-12345",
        "pid": 12345,
        "port": 30001,
        "nasPath": "/nas/tenant-001/sb-12345",
        "password": "uuid-password",
        "status": "running",
        "createdAt": "2026-03-14T10:30:00Z",
        "lastActiveAt": "2026-03-14T12:45:30Z"
    },
    {
        "domainID": "tenant-002",
        "sandboxID": "sb-67890",
        "pid": 67890,
        "port": 30002,
        "nasPath": "/nas/tenant-002/sb-67890",
        "password": "uuid-password",
        "status": "running",
        "createdAt": "2026-03-14T11:00:00Z",
        "lastActiveAt": "2026-03-14T10:00:00Z"
    }
]
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| timeoutSandboxes | List[Sandbox] | 超时沙箱列表 |

**出参示例**:
```python
[
    {
        "domainID": "tenant-002",
        "sandboxID": "sb-67890",
        "pid": 67890,
        "port": 30002,
        "nasPath": "/nas/tenant-002/sb-67890",
        "password": "uuid-password",
        "status": "running",
        "createdAt": "2026-03-14T11:00:00Z",
        "lastActiveAt": "2026-03-14T10:00:00Z"
    }
]
```

**内部处理**:
1. 获取当前时间: `now = datetime.now()`
2. 遍历所有沙箱
3. 对每个沙箱计算空闲时长:
   - `idleTime = now - sandbox.lastActiveAt`
   - `idleSeconds = idleTime.total_seconds()`
4. 判断是否超时:
   - 从 ConfigManager 获取 `idleTimeout` 配置(默认 3600 秒)
   - 若 `idleSeconds > idleTimeout`,则标记为超时
5. 收集所有超时沙箱
6. 返回超时沙箱列表

### 4.4 LogReader 模块

**职责**: 日志读取,读取沙箱内 OpenCode 的日志文件

**参与流程**: 1 个流程 (3.2.3)

---

#### 4.4.1 readLogs

**功能描述**: 读取沙箱内 OpenCode 的日志文件,返回最近 N 行日志内容

**流程**: 3.2.3 - 管理接口：查询单个沙箱

**入参**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| nasPath | str | 是 | NAS 目录路径 |
| limit | int | 否 | 返回日志行数,默认 50 |

**入参示例**:
```python
nasPath = "/nas/tenant-001/sb-12345"
limit = 100
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| logs | List[str] | 日志行列表,按时间倒序 |

**出参示例**:
```python
[
    "2026-03-14 12:45:30 [INFO] Request received: POST /api/chat",
    "2026-03-14 12:45:31 [INFO] Processing message: Hello",
    "2026-03-14 12:45:32 [INFO] Response sent: Hi! How can I help you?",
    "2026-03-14 12:46:00 [INFO] Health check passed",
    "2026-03-14 12:47:00 [INFO] Idle timeout check: last active 60s ago"
]
```

**内部处理**:
1. 构建日志文件路径:
   - `logPath = "{nasPath}/defaultProject/logs/opencode.log"`
2. 检查文件是否存在
3. 若文件不存在,返回空列表 `[]`
4. 若文件存在:
   - 打开文件
   - 读取文件末尾 N 行(使用 tail 或逐行读取)
   - 解析为字符串列表
5. 返回日志行列表

**异常处理**:
- 文件不存在: 返回空列表 `[]`
- 文件读取权限错误: 抛出 `IOError`
- 文件损坏: 尽可能读取可读内容,忽略损坏部分
