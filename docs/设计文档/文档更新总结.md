### 4. 实现细节明确

根据用户解答,明确了以下实现细节:

#### 4.1 模块实例化
- **单例模式**: 所有基础设施模块采用单例模式
- **依赖注入**: 通过 `模块类.get_instance()` 获取实例
- **控制层**: Router Layer (Layer 0) 和 Bootstrap Layer (Layer 5) 作为控制层

#### 4.2 线程安全
- **需加锁**: 端口分配、沙箱创建/销毁、内存字典操作
- **无需加锁**: Redis 操作、只读配置、健康检查、资源监控
- **锁实现**: 全局锁 + 细粒度沙箱锁

#### 4.3 异常处理
- **处理层级**: 接口内部处理异常
- **重试策略**: 失败重试 3 次
- **回滚策略**: 创建失败时回收端口、删除目录、清理 Redis 元数据

#### 4.4 日志规范
- **标准格式**: `{时间} [{日志级别}] [{线程名}] {文件名}:{行号} - {日志信息}`
- **关键操作**: 启动、创建、销毁、重建、配置变更
- **日志级别**: DEBUG, INFO, WARNING, ERROR, CRITICAL

#### 4.5 配置管理
- **加载时机**: Bootstrap Layer 初始化时加载
- **同步机制**: ConfigSyncTask 每 300 秒从 Redis 同步
- **访问方式**: 所有模块通过 ConfigManager 访问内存配置

#### 4.6 测试策略
- **无需单元测试**: 不编写单元测试
- **外部依赖**: 开发环境部署真实 Redis,Bubblewrap 是系统命令
- **测试方式**: HTTP 接口测试
