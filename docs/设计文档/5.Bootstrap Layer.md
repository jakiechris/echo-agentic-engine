# Bootstrap Layer (启动层)

## 层级定位

Bootstrap Layer 负责 Engine 进程启动时的初始化工作,包括配置加载、模块初始化和定时任务注册。

## 核心功能

1. **启动初始化**: 加载 Redis 配置,初始化各模块
2. **任务调度**: 注册定时任务,触发后台巡检
3. **配置同步**: 配置变更时通知相关模块

## 触发的流程

Bootstrap Layer 触发 **9大流程中的后4个流程**(流程 3.2.6 - 3.2.9),这些流程与系统启动和后台任务相关:

| 流程编号 | 流程名称 | 触发模块 | 触发时机 | 触发方式 |
|---------|---------|---------|---------|---------|
| **3.2.6** | Engine 启动主流程 | EngineBootstrap | Engine 进程启动时 | 系统启动时主动执行 |
| **3.2.7** | HealthCheckTask 调用链 | TaskScheduler | 每 30 秒 | 定时任务周期性触发 |
| **3.2.8** | IdleCleanupTask 调用链 | TaskScheduler | 每 60 秒 | 定时任务周期性触发 |
| **3.2.9** | ConfigSyncTask 调用链 | TaskScheduler | 每 300 秒 | 定时任务周期性触发 |

**流程触发特点**:
- ✅ **系统驱动**: 流程由系统内部自动触发,无需外部调用
- ✅ **周期执行**: 后台任务按固定间隔周期性执行
- ✅ **独立运行**: 与 HTTP 请求处理独立,不依赖外部触发

**触发层次**:
```
EngineBootstrap.initialize() 启动时触发:
├─ 流程 3.2.6: Engine 启动主流程 (直接触发)
│   ├─ 建立 Redis 连接
│   ├─ 加载配置
│   ├─ 启动 FastAPI (注册 Router Layer 路由)
│   └─ 注册定时任务
│       └─ TaskScheduler.registerTask()
│           ├─ HealthCheckTask (间隔 30秒)
│           ├─ IdleCleanupTask (间隔 60秒)
│           └─ ConfigSyncTask (间隔 300秒)
│
└─ TaskScheduler.startScheduler() 启动后定时触发:
    ├─ 流程 3.2.7: HealthCheckTask 每 30秒触发
    ├─ 流程 3.2.8: IdleCleanupTask 每 60秒触发
    └─ 流程 3.2.9: ConfigSyncTask 每 300秒触发
```

**未触发的流程**:

- ❌ 流程 3.2.2 - 管理接口：列出所有沙箱 (由 Router Layer 触发)
- ❌ 流程 3.2.3 - 管理接口：查询单个沙箱 (由 Router Layer 触发)

- ❌ 流程 3.2.5 - 管理接口：销毁沙箱 (由 Router Layer 触发)

## 调用关系

- **被调用**: Engine 进程启动时执行 (main.py 入口)
- **调用下游**:
  - Infrastructure Layer: RedisClient (加载配置)
  - Core Layer: ConfigManager (初始化配置)
  - Background Tasks: 注册定时任务
- **横向依赖**: TaskScheduler 与 Background Tasks 协作

## 设计原则

- 启动失败快速退出: 初始化失败时立即终止进程
- 依赖顺序: 先初始化 Infrastructure,再初始化 Core
- 幂等性: 重复执行不会产生副作用

## 模块设计

### 5.1 EngineBootstrap 模块

**职责**: Engine 启动初始化,协调各模块完成启动流程

**参与流程**: 1 个流程 (3.2.6)

---

#### 5.1.1 initialize

**功能描述**: Engine 启动初始化入口，协调各模块完成启动流程，包括建立 Redis 连接、加载配置、启动 FastAPI 应用、注册并启动定时任务，使 Engine 进入运行状态

**流程**: 3.2.6 - Engine 启动主流程

**入参**: 无

**出参**: 无返回值

**出参示例**:
```python
None
```

**内部调用流程**:
1. RedisClient.connect() → 建立 Redis 连接池，测试连接可用性
2. ConfigManager.loadConfig() → 从 Redis 加载配置，若 Redis 中无配置则使用内置默认值
3. 启动 FastAPI 应用 → 注册 API 路由、注册中间件（日志、异常处理）
4. TaskScheduler.registerTask() → 注册定时任务：
   - HealthCheckTask (间隔: Config.healthCheckInterval，默认 30 秒)
   - IdleCleanupTask (间隔: Config.idleCheckInterval，默认 60 秒)
   - ConfigSyncTask (间隔: Config.configSyncInterval，默认 300 秒)
5. TaskScheduler.startScheduler() → 创建后台线程，启动所有定时任务
6. Engine 进入运行状态

**异常**: 初始化失败时立即终止进程并报错（启动失败快速退出原则）

---

### 5.2 TaskScheduler 模块

**职责**: 定时任务调度器,管理后台定时任务的注册和执行

**参与流程**: 2 个流程 (3.2.6, 3.2.9)

---

#### 5.2.1 registerTask

**功能描述**: 注册定时任务，记录任务对象和执行间隔，任务将在调用 startScheduler() 后按指定间隔循环执行

**流程**: 3.2.6 - Engine 启动主流程

**入参**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| task | Task | 是 | 任务对象，包含任务名称和执行方法 |
| interval | int | 是 | 执行间隔时间（秒） |

**入参示例**:
```python
# 注册健康检查任务
task = HealthCheckTask()  # 任务对象
interval = 30  # 30 秒执行一次

# 注册空闲清理任务
task = IdleCleanupTask()
interval = 60  # 60 秒执行一次

# 注册配置同步任务
task = ConfigSyncTask()
interval = 300  # 300 秒执行一次
```

**出参**: 无返回值

**出参示例**:
```python
None
```

**内部处理**:
1. 记录任务对象和执行间隔到内部任务列表
2. 为每个任务分配唯一标识 taskId（如 "health-check-001"）

---

#### 5.2.2 startScheduler

**功能描述**: 启动任务调度器，为每个已注册任务创建独立的后台线程，启动所有定时任务循环执行

**流程**: 3.2.6 - Engine 启动主流程

**入参**: 无

**出参**: 无返回值

**出参示例**:
```python
None
```

**内部处理**:
1. 遍历所有已注册任务
2. 为每个任务创建独立的后台线程
3. 每个线程中循环执行任务：
   - 调用 task.run() 执行任务
   - 休眠 interval 秒
   - 循环执行
4. 所有任务进入运行状态

**线程模型**:
- 每个定时任务在独立线程中运行
- 线程名称：task-{taskId}
- 线程类型：Daemon 线程（随主进程退出）

---

#### 5.2.3 stopScheduler

**功能描述**: 停止所有定时任务，设置停止标志位，等待所有任务线程退出，用于 Engine 关闭时优雅停止后台任务

**流程**: Engine 关闭流程（非标准流程编号）

**入参**: 无

**出参**: 无返回值

**出参示例**:
```python
None
```

**内部处理**:
1. 设置停止标志位 stop_flag = True
2. 通知所有任务线程停止
3. 等待所有任务线程退出（join）
4. 记录关闭日志

**使用场景**: Engine 进程关闭时调用，确保后台任务优雅停止

---

#### 5.2.4 triggerTask

**功能描述**: 立即触发指定任务执行一次，不等待定时器到期，用于配置变更时立即触发相关任务

**流程**: 3.2.9 - ConfigSyncTask 调用链

**入参**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| taskName | str | 是 | 任务名称，如 "IdleCleanupTask" |

**入参示例**:
```python
# 配置变更时立即触发空闲清理任务
taskName = "IdleCleanupTask"
```

**出参**: 无返回值

**出参示例**:
```python
None
```

**内部处理**:
1. 根据任务名称查找任务对象
2. 在调用线程中立即执行 task.run()
3. 不影响定时任务的正常执行周期

**使用场景**:
- ConfigSyncTask 检测到 idleTimeout 减少时，立即触发 IdleCleanupTask 执行
- 需要立即执行某项后台巡检任务时
