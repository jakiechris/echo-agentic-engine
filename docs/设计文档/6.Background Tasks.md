# 6. Background Tasks

## 概述

Echo Agentic Engine 后台任务系统负责定时执行关键维护操作。当前仅保留一个核心任务: ConfigSyncTask。

## 任务调度器

**模块位置**: `src/infrastructure_layer/task_scheduler.py`

**职责**:
- 注册和管理后台任务
- 按配置的时间间隔执行任务
- 支持任务的启动和停止

**接口**:
```python
class TaskScheduler:
    def registerTask(self, task: object, interval_seconds: int) -> None
    def startScheduler(self) -> None
    def stopScheduler(self) -> None
```

## ConfigSyncTask

**模块位置**: `src/background_tasks/config_sync_task.py`

**执行间隔**: 5秒 (配置在config.json: tasks.configSyncInterval)

### 核心职责

1. **写入引擎信息到Redis**
   - 引擎URL (http://ip:port/trans)
   - 最大沙箱数配置
   - 当前使用沙箱数
   - 心跳时间戳

2. **健康检查与清理**
   - 并发检查所有沙箱健康状态
   - 杀死不健康沙箱
   - 从Redis删除不健康沙箱记录

3. **同步沙箱信息表**
   - 对比Redis中的沙箱记录与内存鲜活沙箱
   - 删除过期的Redis记录(多退)
   - 添加新的沙箱记录(少补)

### 执行流程

```
ConfigSyncTask.execute()
├─ 1. 获取引擎信息
│   ├─ 引擎URL: 自动获取本机IP + 端口
│   ├─ 最大沙箱数: 从config.json读取
│   ├─ 当前沙箱数: 从内存沙箱管理器获取
│   └─ 时间戳: UTC当前时间
│
├─ 2. 写入Redis引擎信息表
│   └─ engine:{engine_url}
│       ├─ engineUrl
│       ├─ maxSandboxes
│       ├─ usedSandboxes
│       └─ timestamp
│
├─ 3. 健康检查与清理
│   ├─ 获取所有沙箱列表
│   ├─ 并发检查健康状态
│   │   └─ HealthChecker.checkAllHealth()
│   └─ 对不健康沙箱:
│       ├─ SandboxManager.destroySandbox()
│       └─ RedisClient.deleteSandboxInfo()
│
└─ 4. 同步沙箱信息表
    ├─ RedisClient.getSandboxesByEngine(engine_url)
    ├─ 对比内存鲜活沙箱列表
    ├─ 删除Redis中不存在的沙箱记录
    └─ 添加新沙箱到Redis
        └─ sandbox:{domainID}:{sandboxID}
            ├─ domainID
            ├─ sandboxID
            ├─ engineUrl
            ├─ timestamp
            └─ lastRequestTime
```

## 已删除的任务

### HealthCheckTask (已删除)

**删除原因**:
- 功能已被ConfigSyncTask的健康检查部分替代
- ConfigSyncTask每5秒检查健康状态,更及时
- 发现不健康沙箱直接杀死,不再重建(避免反复重建)

### IdleCleanupTask (已删除)

**删除原因**:
- 空闲清理策略应由echo-agent决定,而非Engine
- echo-agent通过调用管理接口3.2.3主动清理空闲沙箱
- Engine专注沙箱管理,不参与清理策略决策

## 任务配置

**config.json**:
```json
{
  "tasks": {
    "configSyncInterval": 5,
    "healthCheckTimeout": 5,
    "maxRetries": 3
  }
}
```

**配置说明**:
- `configSyncInterval`: ConfigSyncTask执行间隔(秒)
- `healthCheckTimeout`: 健康检查超时时间(秒)
- `maxRetries`: 最大重试次数
