## 1. 模块实例化与依赖注入

### 1.1 单实例模块

以下模块采用**单例模式**,全局唯一实例,在 Bootstrap Layer 初始化时创建:

| 模块 | 实例化方式 | 依赖注入方式 | 线程安全性 |
|------|----------|------------|----------|
| **ConfigManager** | 单例模式 | Bootstrap 初始化时创建 | 只读配置,线程安全 |
| **PortAllocator** | 单例模式 | Bootstrap 初始化时创建 | 端口分配操作需加锁 |
| **NasManager** | 单例模式 | Bootstrap 初始化时创建 | 无状态,线程安全 |
| **RedisClient** | 单例模式 | Bootstrap 初始化时创建 | Redis连接池自带线程安全 |
| **SandboxManager** | 单例模式 | Bootstrap 初始化时创建 | 内存操作需加锁 |
| **HealthChecker** | 单例模式 | Bootstrap 初始化时创建 | 无状态,线程安全 |
| **ResourceMonitor** | 单例模式 | Bootstrap 初始化时创建 | 无状态,线程安全 |
| **IdleMonitor** | 单例模式 | Bootstrap 初始化时创建 | 无状态,线程安全 |
| **LogReader** | 单例模式 | Bootstrap 初始化时创建 | 无状态,线程安全 |
| **TaskScheduler** | 单例模式 | Bootstrap 初始化时创建 | 内部管理任务线程 |
| **RequestProxy** | 单例模式 | Bootstrap 初始化时创建 | 无状态,线程安全 |
| **RequestValidator** | 单例模式 | Bootstrap 初始化时创建 | 无状态,线程安全 |
| **ResponseBuilder** | 单例模式 | Bootstrap 初始化时创建 | 无状态,线程安全 |

**单例实现示例**:

```python
class ConfigManager:
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance._config = None
        return cls._instance

    @classmethod
    def get_instance(cls):
        if cls._instance is None:
            cls._instance = cls()
        return cls._instance
```

### 1.2 非单实例模块

以下模块**每次请求创建新实例**:

| 模块 | 实例化方式 | 说明 |
|------|----------|------|
| **HealthCheckTask** | 每次定时触发创建实例 | 后台任务独立实例 |
| **IdleCleanupTask** | 每次定时触发创建实例 | 后台任务独立实例 |
| **ConfigSyncTask** | 每次定时触发创建实例 | 后台任务独立实例 |

### 1.3 依赖注入方式

**控制层调用模块**:

- **Layer 0 (Router Layer)**: 作为 HTTP 请求入口,调用 Layer 1 (API Layer) 模块
- **Layer 5 (Bootstrap Layer)**: 作为系统启动入口,初始化所有单例模块,启动定时任务

**模块间依赖获取**:

所有单例模块通过 `模块类.get_instance()` 获取实例:


## 2. 线程安全与并发控制

### 2.1 FastAPI 并发模型

FastAPI 使用 **async/await 协程**处理并发请求,每个 HTTP 请求在独立协程中处理。

**协程与线程的关系**:
- FastAPI 运行在单线程事件循环中
- 多个协程并发执行,但在同一线程内切换
- 阻塞操作(如文件 I/O、进程启动)需使用线程池执行

### 2.2 需要加锁的操作

以下操作需要**互斥锁**保护:

| 操作 | 加锁原因 | 锁粒度 |
|------|---------|--------|
| **端口分配** | 防止多个协程分配相同端口 | 全局锁 `PortAllocator._lock` |
| **沙箱创建** | 防止重复创建同一沙箱 | 按 `domainID:sandboxID` 细粒度锁 |
| **沙箱销毁** | 防止销毁正在使用的沙箱 | 按 `domainID:sandboxID` 细粒度锁 |
| **内存沙箱字典操作** | 防止并发修改内存字典 | 全局锁 `SandboxManager._dict_lock` |

**锁实现示例**:

```python
import threading

class PortAllocator:
    _instance = None
    _lock = threading.Lock()  # 端口分配锁

    def allocatePort(self):
        with self._lock:
            # 原子操作: 查询已占用端口,分配新端口
            allocated_ports = RedisClient.get_instance().queryAllocatedPorts()
            port = self._find_available_port(allocated_ports)
            return port

class SandboxManager:
    _instance = None
    _dict_lock = threading.Lock()  # 内存字典锁
    _sandbox_locks = {}  # 细粒度沙箱锁字典
    _lock_mutex = threading.Lock()  # 保护 _sandbox_locks 字典本身

    def createSandbox(self, domainID, sandboxID):
        # 获取沙箱级别的细粒度锁
        lock_key = f"{domainID}:{sandboxID}"
        with self._lock_mutex:
            if lock_key not in self._sandbox_locks:
                self._sandbox_locks[lock_key] = threading.Lock()
            lock = self._sandbox_locks[lock_key]

        with lock:
            # 执行沙箱创建流程
            ...
```

### 2.3 无需加锁的操作

以下操作**无需加锁**:

| 操作 | 原因 |
|------|------|
| **Redis 操作** | Redis 客户端自带线程安全机制,连接池管理并发连接 |
| **只读配置** | ConfigManager 的配置在加载后只读,无需加锁 |
| **健康检查** | 无状态操作,每次独立执行 |
| **资源监控** | 无状态操作,读取 `/proc` 文件系统 |

### 2.4 Background Tasks 并发

3 个后台任务在独立线程中执行,**相互解耦,无竞争条件**:

- **HealthCheckTask**: 只读 Redis,读写沙箱进程,无需与 IdleCleanupTask 同步
- **IdleCleanupTask**: 只读 Redis,销毁沙箱,与 HealthCheckTask 操作不同沙箱
- **ConfigSyncTask**: 只读 Redis,更新内存配置,独立执行

**访问共享资源时**:
- 访问 Redis: 无需加锁 (Redis 线程安全)
- 访问内存沙箱字典: 需加锁 (使用 `SandboxManager._dict_lock`)

## 3. 内存数据结构设计

### 3.1 Sandbox 对象存储

**存储结构**: Python 字典

```python
class SandboxManager:
    _instance = None
    _sandboxes = {}  # 内存沙箱字典: {(domainID, sandboxID): Sandbox}

    def getSandbox(self, domainID, sandboxID):
        with self._dict_lock:
            return self._sandboxes.get((domainID, sandboxID))

    def _add_sandbox(self, sandbox):
        with self._dict_lock:
            key = (sandbox['domainID'], sandbox['sandboxID'])
            self._sandboxes[key] = sandbox

    def _remove_sandbox(self, domainID, sandboxID):
        with self._dict_lock:
            key = (domainID, sandboxID)
            if key in self._sandboxes:
                del self._sandboxes[key]
```

**Sandbox 对象定义**:

```python
from dataclasses import dataclass
from typing import Dict, Optional

@dataclass
class Sandbox:
    domainID: str
    sandboxID: str
    pid: int
    port: int
    nasPath: str
    password: str
    status: str  # 'running', 'destroying', 'error'
    createdAt: str  # ISO 8601
    lastActiveAt: str  # ISO 8601
```

### 3.2 查询效率

- `getSandbox(domainID, sandboxID)`: 字典查找 O(1),简单高效
- `listAllSandboxes()`: 返回字典所有值,简单遍历

**无需优化查询效率**: 因为是人工介入管理,沙箱数量有限 (默认最多 50 个)。

## 4. 异常处理策略

### 4.1 异常处理层级

**接口内部处理异常**,便于快速定位问题根源:

```python
class SandboxManager:
    def createSandbox(self, domainID, sandboxID):
        try:
            # 1. 分配端口
            port = PortAllocator.get_instance().allocatePort()
        except Exception as e:
            logger.error(f"端口分配失败: {e}")
            raise PortAllocationError(f"端口分配失败: {e}")

        try:
            # 2. 准备 NAS 目录
            nasPath = NasManager.get_instance().prepareDirectory(domainID, sandboxID)
        except Exception as e:
            logger.error(f"NAS 目录创建失败: {e}")
            # 回滚: 释放端口
            PortAllocator.get_instance().recyclePort(port)
            raise NasPreparationError(f"NAS 目录创建失败: {e}")

        try:
            # 3. 启动沙箱进程
            pid, password = BubblewrapLauncher.get_instance().launchSandbox(nasPath, port)
        except Exception as e:
            logger.error(f"沙箱进程启动失败: {e}")
            # 回滚: 释放端口,删除目录
            PortAllocator.get_instance().recyclePort(port)
            NasManager.get_instance().removeDirectory(domainID, sandboxID)
            raise SandboxLaunchError(f"沙箱进程启动失败: {e}")
```

### 4.2 重试策略

**重试 3 次**后报错:

```python
import time

def retry_operation(operation, max_retries=3, delay=1):
    for attempt in range(max_retries):
        try:
            return operation()
        except Exception as e:
            if attempt == max_retries - 1:
                raise
            logger.warning(f"操作失败,重试 {attempt + 1}/{max_retries}: {e}")
            time.sleep(delay)

# 使用示例
redis_client = RedisClient.get_instance()
result = retry_operation(lambda: redis_client.setSandboxMetadata(...))
```

### 4.3 回滚策略

**沙箱创建失败时的回滚**:

| 已执行步骤 | 回滚操作 |
|-----------|---------|
| 端口分配 | 调用 `PortAllocator.recyclePort(port)` |
| NAS 目录创建 | 调用 `NasManager.removeDirectory(domainID, sandboxID)` |
| Redis 元数据写入 | 调用 `RedisClient.deleteSandboxMetadata(domainID, sandboxID)` |
| 进程启动失败 | 自动终止进程 (BubblewrapLauncher 内部处理) |

**示例**:

```python
def createSandbox(self, domainID, sandboxID):
    port = None
    nasPath = None
    metadata_written = False

    try:
        # 1. 分配端口
        port = PortAllocator.get_instance().allocatePort()

        # 2. 准备 NAS 目录
        nasPath = NasManager.get_instance().prepareDirectory(domainID, sandboxID)

        # 3. 启动进程
        pid, password = BubblewrapLauncher.get_instance().launchSandbox(nasPath, port)

        # 4. 写入元数据
        RedisClient.get_instance().setSandboxMetadata(domainID, sandboxID, {...})
        metadata_written = True

        # 5. 添加到内存
        sandbox = Sandbox(...)
        self._add_sandbox(sandbox)

        return sandbox

    except Exception as e:
        logger.error(f"沙箱创建失败: {e}")

        # 回滚
        if metadata_written:
            RedisClient.get_instance().deleteSandboxMetadata(domainID, sandboxID)
        if nasPath:
            NasManager.get_instance().removeDirectory(domainID, sandboxID)
        if port:
            PortAllocator.get_instance().recyclePort(port)

        raise
```

```python
import logging

        datefmt='%Y-%m-%d %H:%M:%S'

    # 控制台处理器
    console_handler = logging.StreamHandler(sys.stdout)
    console_handler.setFormatter(formatter)

    # 根日志器
    logger = logging.getLogger()

    logger.addHandler(console_handler)
    logger.addHandler(file_handler)

    return logger

logger = setup_logging()
```

### 6.2 配置访问方式

**所有模块通过 ConfigManager 访问配置**:

```python
class SandboxManager:
    def createSandbox(self, domainID, sandboxID):
        # 获取配置
        config = ConfigManager.get_instance().getConfig()
        max_sandboxes = config['maxSandboxes']

        # 检查沙箱数量限制
        current_count = len(self.listAllSandboxes())
        if current_count >= max_sandboxes:
            raise MaxSandboxesExceededError(f"已达到最大沙箱数量限制: {max_sandboxes}")
```

**ConfigManager 实现**:

```python
class ConfigManager:
    _instance = None
    _config = None
    _lock = threading.Lock()

    def loadConfig(self):
        """从 Redis 加载配置到内存"""
        with self._lock:
            redis_config = RedisClient.get_instance().fetchConfig()
            if redis_config:
                self._config = self._merge_with_defaults(redis_config)
            else:
                self._config = self._get_default_config()

    def getConfig(self):
        """获取当前配置 (每次都从内存读取最新值)"""
        with self._lock:
            return self._config.copy()

    def updateConfig(self, new_config):
        """更新内存配置"""
        with self._lock:
            self._config = new_config
```

## 7. FastAPI 路由与模块绑定

### 7.1 Router Layer 实现

**Layer 0 作为控制层,调用 Layer 1 (API Layer) 模块**:

```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse

app = FastAPI()

# 获取单例实例
request_validator = RequestValidator.get_instance()
response_builder = ResponseBuilder.get_instance()
sandbox_manager = SandboxManager.get_instance()
request_proxy = RequestProxy.get_instance()

@app.api_route("/trans/{path:path}", methods=["GET", "POST", "PUT", "DELETE"])
async def proxy_request(request: Request, path: str):
    """OpenCode API 代理接口"""
    try:
        # 1. 验证请求 (调用 API Layer)
        headers = dict(request.headers)
        domainID, sandboxID = request_validator.validateProxyRequest(headers)

        # 2. 获取沙箱 (调用 Core Layer)
        sandbox = sandbox_manager.getSandbox(domainID, sandboxID)
        if not sandbox:
            sandbox = sandbox_manager.createSandbox(domainID, sandboxID)
        else:
            # 健康检查
            health = HealthChecker.get_instance().checkHealth(sandbox)
            if health == 'unhealthy':
                sandbox = sandbox_manager.rebuildSandbox(domainID, sandboxID)

        # 3. 代理请求 (调用 Core Layer)
        method = request.method
        body = await request.json() if request.method in ["POST", "PUT"] else None
        response = request_proxy.proxyRequest(sandbox, method, path, headers, body)

        # 4. 更新活跃时间
        RedisClient.get_instance().updateLastActive(domainID, sandboxID)

        # 5. 返回响应
        return response_builder.buildSuccessResponse(response.json())

    except ValidationError as e:
        return response_builder.buildErrorResponse(e.code, e.message, e.details)
    except Exception as e:
        logger.error(f"代理请求失败: {e}", exc_info=True)
        return response_builder.handleException(e)

@app.post("/admin/sandbox/list")
async def list_sandboxes(request: Request):
    """列出所有沙箱"""
    try:
        # 1. 验证请求
        body = await request.json()
        request_validator.validateAdminListRequest(body)

        # 2. 获取沙箱列表
        sandboxes = sandbox_manager.listAllSandboxes()

        # 3. 返回响应
        return response_builder.buildSuccessResponse({
            "total": len(sandboxes),
            "sandboxes": sandboxes
        })

    except Exception as e:
        logger.error(f"列出沙箱失败: {e}", exc_info=True)
        return response_builder.handleException(e)
```
