# Infrastructure Layer (基础设施层)

## 层级定位

Infrastructure Layer 为 Engine 提供底层基础设施服务,包括端口分配、文件系统管理、进程管理和存储访问。

## 核心功能

1. **端口服务**: 分配和回收沙箱端口 (30001-40000)
2. **存储服务**: 准备 NAS 共享目录,管理文件系统
3. **进程服务**: 启动和终止 Bubblewrap 沙箱进程
4. **存储访问**: Redis 客户端,读写元数据和配置

## 调用关系

- **被调用**: 由 Core Layer 调用
- **调用外部**:
  - 系统 API: 端口分配、进程管理
  - 文件系统: NAS 目录操作
  - Redis: 外部存储服务
- **不调用上层**: 不依赖 API Layer, Core Layer

## 设计原则

- 资源池化: 端口、进程等资源统一管理
- 隔离性: 每个模块独立封装底层细节
- 线程安全: 多沙箱并发场景下保证资源分配安全

## 模块设计

### 3.1 PortAllocator 模块

**职责**: 端口分配与回收管理

**参与流程**: 3 个流程 (3.2.1, 3.2.4, 3.2.5)

#### 接口定义

| 方法名 | 流程 | 入参 | 出参 | 说明 |
|--------|------|------|------|------|
| `allocatePort` | 3.2.1, 3.2.4 | 无 | `int` | 查询已占用端口,在端口范围 30001-40000 内随机选择可用端口 |
| `recyclePort` | 3.2.5 | `port: int` | `bool` | 从 Redis 已分配端口集合中移除端口 |
| `getAllocatedPorts` | 3.2.1, 3.2.4 | 无 | `List[int]` | 查询 Redis 获取所有已分配端口 |

---

### 3.2 NasManager 模块

**职责**: NAS 目录管理,为沙箱准备文件存储环境

**参与流程**: 2 个流程 (3.2.1, 3.2.4)

#### 接口定义

| 方法名 | 流程 | 入参 | 出参 | 说明 |
|--------|------|------|------|------|
| `prepareDirectory` | 3.2.1, 3.2.4 | `domainID: str, sandboxID: str` | `str` | 创建 `/nas/{domainID}/{sandboxID}/`、`defaultProject/`、`.config/.opencode/` 目录,设置权限,返回 NAS 路径 |
| `removeDirectory` | - | `domainID: str, sandboxID: str` | `bool` | 删除 `/nas/{domainID}/{sandboxID}/` 目录 (目前未使用) |

---

### 3.3 BubblewrapLauncher 模块

**职责**: Bubblewrap 进程启动,隔离运行 OpenCode Serve

**参与流程**: 3 个流程 (3.2.1, 3.2.4, 3.2.5)

#### 接口定义

| 方法名 | 流程 | 入参 | 出参 | 说明 |
|--------|------|------|------|------|
| `launchSandbox` | 3.2.1, 3.2.4 | `nasPath: str, port: int` | `(int, str)` | 生成随机密码、构建 bwrap 命令 (`--unshare-net --unshare-pid --bind {nasPath} /workspace --port {port}:4096`)、启动进程、等待就绪,返回 (pid, password) |
| `killSandbox` | 3.2.5 | `pid: int` | `bool` | 发送 SIGTERM 信号,等待进程退出(10秒超时),若未退出则发送 SIGKILL 信号 |

---

### 3.4 RedisClient 模块

**职责**: Redis 客户端,管理沙箱元数据和配置

**参与流程**: 7 个流程 (3.2.1 - 3.2.6, 3.2.9)

#### 接口定义

| 方法名 | 流程 | 入参 | 出参 | 说明 |
|--------|------|------|------|------|
| `connect` | 3.2.6 | 无 | `bool` | 建立 Redis 连接池,测试连接可用性 |
| `setSandboxMetadata` | 3.2.1, 3.2.4 | `domainID: str, sandboxID: str, metadata: Dict` | `bool` | 写入沙箱元数据到 `sandbox:{domainID}:{sandboxID}` (Hash) |
| `getSandboxMetadata` | 3.2.3 | `domainID: str, sandboxID: str` | `Metadata` | 查询沙箱元数据 |
| `getBatchMetadata` | 3.2.2 | `sandboxIDs: List[str]` | `Dict[str, Metadata]` | 批量查询沙箱元数据 |
| `updateMetadata` | 3.2.1, 3.2.5 | `domainID: str, sandboxID: str, **kwargs` | `bool` | 更新指定字段 |
| `deleteSandboxMetadata` | 3.2.5 | `domainID: str, sandboxID: str` | `bool` | 删除沙箱元数据 Key |
| `updateLastActive` | 3.2.1 | `domainID: str, sandboxID: str` | `bool` | 更新 lastActiveAt 字段为当前时间 |
| `getAllocatedPorts` | 3.2.1, 3.2.4 | 无 | `List[int]` | 查询所有沙箱元数据,提取 port 字段 |
| `getConfig` | 3.2.6, 3.2.9 | 无 | `Dict` | 查询配置 `HGETALL engine:config` |

**Redis Key 设计**:
- `sandbox:{domainID}:{sandboxID}` (Hash): 沙箱元数据
  - 字段: pid, port, status, createdAt, lastActiveAt, password
- `engine:config` (Hash): Engine 全局配置
  - 字段: maxSandboxes, idleTimeout, healthCheckInterval 等
