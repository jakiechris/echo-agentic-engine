# Infrastructure Layer (基础设施层)

## 层级定位

Infrastructure Layer 为 Engine 提供底层基础设施服务,包括端口分配、文件系统管理、进程管理和存储访问。

## 核心功能

1. **端口服务**: 分配和回收沙箱端口 (30001-40000)
2. **存储服务**: 准备 NAS 共享目录,管理文件系统
3. **进程服务**: 启动和终止 Bubblewrap 沙箱进程
4. **存储访问**: Redis 客户端,读写元数据和配置

## 包含模块

| 模块名称 | 职责 | 参与流程数 |
|---------|------|----------|
| **PortAllocator** | 端口分配与回收 | 3 个流程 (3.2.1, 3.2.4, 3.2.5) |
| **NasManager** | NAS 目录管理 | 2 个流程 (3.2.1, 3.2.4) |
| **BubblewrapLauncher** | Bubblewrap 进程启动 | 3 个流程 (3.2.1, 3.2.4, 3.2.5) |
| **RedisClient** | Redis 客户端 | 7 个流程 (3.2.1 - 3.2.6, 3.2.9) |

## 调用关系

- **被调用**: 由 Core Layer 调用
- **调用外部**:
  - 系统 API: 端口分配、进程管理
  - 文件系统: NAS 目录操作
  - Redis: 外部存储服务
- **不调用上层**: 不依赖 API Layer, Core Layer

## 设计原则

- 资源池化: 端口、进程等资源统一管理
- 隔离性: 每个模块独立封装底层细节
- 线程安全: 多沙箱并发场景下保证资源分配安全
