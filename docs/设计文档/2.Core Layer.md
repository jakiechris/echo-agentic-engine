# Core Layer (核心层)

## 层级定位

Core Layer 是 Engine 的业务逻辑核心,负责协调各模块完成沙箱生命周期管理、请求代理、配置管理等核心业务流程。

## 核心功能

1. **沙箱管理**: 创建、销毁、重建沙箱,协调基础设施模块
2. **请求代理**: 将业务请求代理到沙箱内 OpenCode Serve
3. **配置管理**: 加载和更新配置,处理配置变更

## 调用关系

- **被调用**: 由 API Layer 调用
- **调用下游**:
  - Infrastructure Layer: PortAllocator, NasManager, BubblewrapLauncher, RedisClient
  - Monitoring Layer: HealthChecker, ResourceMonitor, IdleMonitor
- **横向依赖**: Core Layer 内部模块可能相互调用

## 设计原则

- 单一职责: 每个模块职责单一,边界清晰
- 协调者模式: SandboxManager 作为核心协调者,不直接操作资源
- 异步优先: 使用 FastAPI 异步特性,避免阻塞调用

## 模块设计

### 2.1 SandboxManager 模块

**职责**: 沙箱生命周期管理的核心协调者,协调各模块完成沙箱的创建、销毁、重建

**参与流程**: 7 个流程 (3.2.1 - 3.2.5, 3.2.7, 3.2.8)

#### 接口定义

| 方法名 | 流程 | 入参 | 出参 | 说明 |
|--------|------|------|------|------|
| `getSandbox` | 3.2.1 | `domainID: str, sandboxID: str` | `Sandbox \| None` | 从内存查找沙箱实例,不存在返回 None,存在则检查健康,不健康则自动重建 |
| `listAllSandboxes` | 3.2.2, 3.2.7, 3.2.8 | 无 | `List[Sandbox]` | 从内存获取所有沙箱实例 |
| `createSandbox` | 3.2.4 | `domainID: str, sandboxID: str` | `Sandbox` | 协调各模块创建沙箱: 分配端口、准备目录、启动进程、写入元数据 |
| `destroySandbox` | 3.2.5 | `domainID: str, sandboxID: str` | `bool` | 协调各模块销毁沙箱: 更新状态、终止进程、回收端口、删除元数据、移除内存实例 |
| `rebuildSandbox` | 3.2.1 | `domainID: str, sandboxID: str` | `Sandbox` | 重建沙箱: 先销毁旧沙箱,再创建新沙箱 |
| `batchRebuild` | 3.2.7 | `sandboxes: List[Sandbox]` | `List[Sandbox]` | 批量重建沙箱 |
| `batchDestroy` | 3.2.8 | `sandboxes: List[Sandbox]` | `List[bool]` | 批量销毁沙箱 |

### 2.2 RequestProxy 模块

**职责**: HTTP 请求代理,将请求转发到沙箱内 OpenCode Serve

**参与流程**: 1 个流程 (3.2.1)

#### 接口定义

| 方法名 | 流程 | 入参 | 出参 | 说明 |
|--------|------|------|------|------|
| `proxyRequest` | 3.2.1 | `sandbox: Sandbox, method: str, path: str, headers: Dict, body: Dict` | `Response` | 路径转换、Headers 转换、构建请求 URL、发送 HTTP 请求到沙箱、返回 OpenCode 原生响应 |

---

### 2.3 ConfigManager 模块

**职责**: 配置管理,从 Redis 加载和管理 Engine 配置

**参与流程**: 2 个流程 (3.2.6, 3.2.9)

#### 接口定义

| 方法名 | 流程 | 入参 | 出参 | 说明 |
|--------|------|------|------|------|
| `loadConfig` | 3.2.6 | 无 | `Config` | 从 Redis 读取配置,若 Redis 中无配置则使用内置默认值,验证配置项合法性 |
| `get` | 3.2.6, 3.2.9 | `key: str` | `Any` | 获取单个配置项 |
| `updateConfig` | 3.2.9 | `changes: Dict` | `bool` | 更新内存中的配置对象 |
| `compareConfig` | 3.2.9 | `local: Config, redis: Config` | `Dict` | 比较本地配置和 Redis 配置,返回差异字段 |

**配置项及默认值**:
- `portRange`: 端口池范围 (30001-40000)
- `idleTimeout`: 空闲超时时长 (3600 秒)
- `healthCheckInterval`: 健康检查间隔 (30 秒)
- `idleCheckInterval`: 空闲检查间隔 (60 秒)
- `configSyncInterval`: 配置同步间隔 (300 秒)
- `maxRetries`: 请求重试次数 (3 次)
- `healthCheckTimeout`: 健康检查超时 (5 秒)
- `maxSandboxes`: 最大沙箱数量 (50)
- `redisHost`: Redis 主机 ("localhost")
- `redisPort`: Redis 端口 (6379)
- `nasRootPath`: NAS 根路径 ("/nas")
- `openCodeCommand`: OpenCode 命令路径 ("/usr/local/bin/opencode")
