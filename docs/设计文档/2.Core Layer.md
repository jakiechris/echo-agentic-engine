# Core Layer (核心层)

## 层级定位

Core Layer 是 Engine 的业务逻辑核心,负责协调各模块完成沙箱生命周期管理、请求代理、配置管理等核心业务流程。

## 核心功能

1. **沙箱管理**: 创建、销毁、重建沙箱,协调基础设施模块
2. **请求代理**: 将业务请求代理到沙箱内 OpenCode Serve
3. **配置管理**: 加载和更新配置,处理配置变更

## 调用关系

- **被调用**: 由 API Layer 调用
- **调用下游**:
  - Infrastructure Layer: PortAllocator, NasManager, BubblewrapLauncher, RedisClient
  - Monitoring Layer: HealthChecker, ResourceMonitor, IdleMonitor
- **横向依赖**: Core Layer 内部模块可能相互调用

## 设计原则

- 单一职责: 每个模块职责单一,边界清晰
- 协调者模式: SandboxManager 作为核心协调者,不直接操作资源
- 异步优先: 使用 FastAPI 异步特性,避免阻塞调用

## 模块设计

### 2.1 SandboxManager 模块

**职责**: 沙箱生命周期管理的核心协调者,协调各模块完成沙箱的创建、销毁、重建

**参与流程**: 7 个流程 (3.2.1 - 3.2.5, 3.2.7, 3.2.8)

---

#### 2.1.1 getSandbox

**功能描述**: 从内存查找沙箱实例，若不存在返回 None，若存在则检查健康状态，不健康则自动重建

**流程**: 3.2.1 - OpenCode API 代理主流程

**入参**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| domainID | str | 是 | 租户标识 |
| sandboxID | str | 是 | 沙箱标识 |

**入参示例**:
```python
domainID = "tenant-001"
sandboxID = "sb-12345"
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| sandbox | Sandbox \| None | 沙箱实例对象，不存在则返回 None |

**出参示例**:
```python
# 存在且健康
{
    "domainID": "tenant-001",
    "sandboxID": "sb-12345",
    "pid": 12345,
    "port": 30001,
    "nasPath": "/nas/tenant-001/sb-12345",
    "password": "uuid-password",
    "status": "running",
    "createdAt": "2026-03-14T10:30:00Z",
    "lastActiveAt": "2026-03-14T12:45:30Z"
}

# 不存在
None
```

**Sandbox 对象字段说明**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| domainID | str | 租户标识 |
| sandboxID | str | 沙箱标识 |
| pid | int | 进程 ID |
| port | int | 宿主机端口 |
| nasPath | str | NAS 目录路径 |
| password | str | OpenCode 访问密码 |
| status | str | 沙箱状态: `running`, `destroying`, `error` |
| createdAt | str | 创建时间 (ISO 8601) |
| lastActiveAt | str | 最后活跃时间 (ISO 8601) |

#### 2.1.3 createSandbox

**功能描述**: 协调各模块创建沙箱，包括分配端口、准备目录、启动进程、写入元数据

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| domainID | str | 是 | 租户标识 |
| sandboxID | str | 是 | 沙箱标识 |

**入参示例**:
```python
domainID = "tenant-001"
sandboxID = "sb-12345"
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| sandbox | Sandbox | 新创建的沙箱实例对象 |

**出参示例**:
```python
{
    "domainID": "tenant-001",
    "sandboxID": "sb-12345",
    "pid": 12345,
    "port": 30001,
    "nasPath": "/nas/tenant-001/sb-12345",
    "password": "uuid-password",
    "status": "running",
```

#### 2.1.4 destroySandbox

**功能描述**: 协调各模块销毁沙箱，包括更新状态、终止进程、回收端口、删除元数据、移除内存实例

**流程**: 3.2.5 - 管理接口：销毁沙箱

**入参**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| domainID | str | 是 | 租户标识 |
| sandboxID | str | 是 | 沙箱标识 |

**入参示例**:
```python
domainID = "tenant-001"
sandboxID = "sb-12345"
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| success | bool | 是否销毁成功 |

**出参示例**:
```python
True
```

**内部调用流程**:
1. RedisClient.updateMetadata(status="destroying") → 更新状态
2. BubblewrapLauncher.killSandbox() → 终止进程
3. PortAllocator.recyclePort() → 回收端口
4. RedisClient.deleteSandboxMetadata() → 删除元数据
5. 从内存移除沙箱实例

---

#### 2.2.1 proxyRequest

**功能描述**: 将 HTTP 请求代理到沙箱内 OpenCode Serve，进行路径转换、Headers 转换、构建请求 URL

**流程**: 3.2.1 - OpenCode API 代理主流程

**入参**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| sandbox | Sandbox | 是 | 沙箱实例对象 |
| method | str | 是 | HTTP 方法: GET, POST, PUT, DELETE |
| path | str | 是 | 请求路径，如 `/api/chat` |
| headers | Dict[str, str] | 是 | HTTP Headers 字典 |
| body | Dict | 否 | HTTP Body (JSON) |

**入参示例**:
```python
sandbox = {
    "domainID": "tenant-001",
    "sandboxID": "sb-12345",
    "port": 30001,
    "password": "uuid-password"
}
method = "POST"
path = "/api/chat"
headers = {
    "Content-Type": "application/json",
    "Authorization": "Bearer xxx"
}
body = {
    "message": "Hello"
}
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| response | Response | HTTP Response 对象 |

**出参示例**:
```python
# HTTP 200
{
    "status": "success",
    "data": {
        "response": "Hello! How can I help you?"
    }
}
```

**内部处理**:
1. 路径转换: `/trans/api/chat` → `/api/chat`
2. Headers 转换: 移除 X-Domain-ID, X-Sandbox-ID
3. 构建请求: `http://127.0.0.1:30001/api/chat`
4. 添加 Authorization: `Bearer {password}`
5. 发送 HTTP 请求并返回响应

**出参示例**:
```python
{
    "portRange": {
        "min": 30001,
        "max": 40000
    },
    "idleTimeout": 3600,
    "healthCheckInterval": 30,
    "idleCheckInterval": 60,
    "configSyncInterval": 300,
    "maxRetries": 3,
    "healthCheckTimeout": 5,
    "maxSandboxes": 50,
    "redisHost": "localhost",
    "redisPort": 6379,
    "redisPassword": "",
    "redisDB": 0,
    "nasRootPath": "/nas",
    "enginePort": 8000,
    "openCodeCommand": "/usr/local/bin/opencode",
    "logLevel": "INFO",
    "logFile": "engine.log"
}
```

**Config 对象字段说明**:

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| portRange | Dict | {min: 30001, max: 40000} | 端口池范围 |
| idleTimeout | int | 3600 | 空闲超时时长 (秒) |
| healthCheckInterval | int | 30 | 健康检查间隔 (秒) |

| configSyncInterval | int | 300 | 配置同步间隔 (秒) |
| maxRetries | int | 3 | 请求重试次数 |
| healthCheckTimeout | int | 5 | 健康检查超时 (秒) |
| maxSandboxes | int | 50 | 最大沙箱数量 |
| redisHost | str | "localhost" | Redis 主机地址 |
| redisPort | int | 6379 | Redis 端口 |
| redisPassword | str | "" | Redis 密码 (可选) |
| redisDB | int | 0 | Redis 数据库编号 |
| nasRootPath | str | "/nas" | NAS 根路径 |
| enginePort | int | 8000 | Engine 监听端口 |
| openCodeCommand | str | "/usr/local/bin/opencode" | OpenCode 命令路径 |
