# API Layer (接口层)

## 层级定位

API Layer 是 Engine 对外提供服务的入口层,负责处理 HTTP 请求和响应,参数验证以及响应格式化。

## 核心功能

1. **请求验证**: 提取并验证 HTTP 请求中的参数 (Headers, Body)
2. **响应构建**: 统一响应格式,处理错误和异常
3. **协议转换**: 将 HTTP 请求转换为内部调用,将内部结果转换为 HTTP 响应

## 调用关系

- **被调用**: 由 FastAPI 路由函数调用
- **调用下游**: Core Layer 的 SandboxManager, ConfigManager 等模块
- **不依赖**: Infrastructure Layer, Monitoring Layer 等底层模块

## 设计原则

- 无状态: 不持有任何运行时状态
- 轻量级: 仅做参数验证和格式转换,不包含业务逻辑
- 统一格式: 所有接口统一使用 `{"status": "...", "data": {...}}` 格式

## 模块设计

### 1.1 RequestValidator 模块

**职责**: 请求参数验证与提取

**参与流程**: 5 个流程 (3.2.1 - 3.2.5)

#### 接口定义

| 方法名 | 流程 | 入参 | 出参 | 说明 |
|--------|------|------|------|------|
| `validateProxyRequest` | 3.2.1 | `headers: Dict[str, str]` | `{domainID: str, sandboxID: str}` | 验证代理请求，从 Headers 提取 domainID 和 sandboxID |
| `validateAdminListRequest` | 3.2.2 | `body: Dict` | `{}` | 验证列出沙箱请求，验证 Body 为空 |
| `validateAdminGetRequest` | 3.2.3 | `body: Dict` | `{domainID: str, sandboxID: str}` | 验证查询单个沙箱请求，从 Body 提取参数 |
| `validateAdminCreateRequest` | 3.2.4 | `body: Dict` | `{domainID: str, sandboxID: str}` | 验证创建沙箱请求，从 Body 提取参数 |
| `validateAdminDestroyRequest` | 3.2.5 | `body: Dict` | `{domainID: str, sandboxID: str}` | 验证销毁沙箱请求，从 Body 提取参数 |

**异常处理**: 所有方法验证失败时抛出 `ValidationError` 异常

### 1.2 ResponseBuilder 模块

**职责**: 统一响应格式构建与错误处理

**参与流程**: 5 个流程 (3.2.1 - 3.2.5)

#### 接口定义

| 方法名 | 流程 | 入参 | 出参 | 说明 |
|--------|------|------|------|------|
| `success` | 3.2.1 - 3.2.5 | `data: Any` | `Response` | 构建成功响应，格式: `{"status": "success", "data": {...}}` |
| `error` | 3.2.1 - 3.2.5 | `code: str, message: str, details: Optional[Dict]` | `Response` | 构建错误响应，格式: `{"status": "error", "error": {"code": "...", "message": "...", "details": {...}}}` |
| `handleException` | 3.2.1 - 3.2.5 | `exception: Exception` | `Response` | 异常统一处理，根据异常类型映射 HTTP 状态码 |
