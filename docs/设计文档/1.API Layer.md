# API Layer (接口层)

## 层级定位

API Layer 是 Engine 对外提供服务的入口层,负责处理 HTTP 请求和响应,参数验证以及响应格式化。

## 核心功能

1. **请求验证**: 提取并验证 HTTP 请求中的参数 (Headers, Body)
2. **响应构建**: 统一响应格式,处理错误和异常
3. **协议转换**: 将 HTTP 请求转换为内部调用,将内部结果转换为 HTTP 响应

## 调用关系

- **被调用**: 由 FastAPI 路由函数调用
- **调用下游**: Core Layer 的 SandboxManager, ConfigManager 等模块
- **不依赖**: Infrastructure Layer, Monitoring Layer 等底层模块

## 设计原则

- 无状态: 不持有任何运行时状态
- 轻量级: 仅做参数验证和格式转换,不包含业务逻辑
- 统一格式: 所有接口统一使用 `{"status": "...", "data": {...}}` 格式

## 模块设计

### 1.1 RequestValidator 模块

**职责**: 请求参数验证与提取

**参与流程**: 5 个流程 (3.2.1 - 3.2.5)

---

#### 1.1.1 validateProxyRequest

**功能描述**: 验证 OpenCode API 代理请求，从 HTTP Headers 中提取租户标识和沙箱标识

**流程**: 3.2.1 - OpenCode API 代理主流程

**入参**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| headers | Dict[str, str] | 是 | HTTP 请求头字典 |

**入参示例**:
```python
{
    "X-Domain-ID": "tenant-001",
    "X-Sandbox-ID": "sb-12345",
    "Content-Type": "application/json",
    "Authorization": "Bearer xxx"
}
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| domainID | str | 租户标识 |
| sandboxID | str | 沙箱标识 |

**出参示例**:
```python
{
    "domainID": "tenant-001",
    "sandboxID": "sb-12345"
}
```

**异常**: 验证失败时抛出 `ValidationError`，错误码 `MISSING_DOMAIN_ID` 或 `MISSING_SANDBOX_ID`

### 1.2 ResponseBuilder 模块

**职责**: 统一响应格式构建与错误处理

**参与流程**: 5 个流程 (3.2.1 - 3.2.5)

---

#### 1.2.1 success

**功能描述**: 构建成功响应，统一格式为 `{"status": "success", "data": {...}}`

**流程**: 3.2.1 - 3.2.5

**入参**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| data | Any | 是 | 成功数据，可以是字典、列表或任意类型 |

**入参示例**:
```python
{
    "domainID": "tenant-001",
    "sandboxID": "sb-12345",
    "status": "running",
    "port": 30001
}
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| (Response对象) | Response | FastAPI Response 对象，包含状态码和 JSON 响应体 |

**出参示例**:
```python
# HTTP 200
{
    "status": "success",
    "data": {
        "domainID": "tenant-001",
        "sandboxID": "sb-12345",
        "status": "running",
        "port": 30001
    }
}
```

---

#### 1.2.2 error

**功能描述**: 构建错误响应，统一格式为 `{"status": "error", "error": {"code": "...", "message": "...", "details": {...}}}`

**流程**: 3.2.1 - 3.2.5

**入参**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| code | str | 是 | 错误码，如 `MISSING_DOMAIN_ID`、`SANDBOX_UNAVAILABLE` |
| message | str | 是 | 错误描述信息 |
| details | Optional[Dict] | 否 | 错误详细信息，可选 |

**入参示例**:
```python
code = "MISSING_DOMAIN_ID"
message = "缺少 X-Domain-ID Header"
details = {"header": "X-Domain-ID"}
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| (Response对象) | Response | FastAPI Response 对象，包含状态码和 JSON 响应体 |

**出参示例**:
```python
# HTTP 400
{
    "status": "error",
    "error": {
        "code": "MISSING_DOMAIN_ID",
        "message": "缺少 X-Domain-ID Header",
        "details": {
            "header": "X-Domain-ID"
        }
    }
}
```

---

#### 1.2.3 handleException

**功能描述**: 异常统一处理，根据异常类型映射 HTTP 状态码，并构建错误响应

**流程**: 3.2.1 - 3.2.5

**入参**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| exception | Exception | 是 | 捕获的异常对象 |

**入参示例**:
```python
ValidationError("缺少 X-Domain-ID Header")
```

**出参**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| (Response对象) | Response | FastAPI Response 对象，包含状态码和 JSON 响应体 |

**出参示例**:
```python
# HTTP 400 (ValidationError)
{
    "status": "error",
    "error": {
        "code": "VALIDATION_ERROR",
        "message": "缺少 X-Domain-ID Header",
        "details": {}
    }
}

# HTTP 503 (SandboxUnavailableError)
{
    "status": "error",
    "error": {
        "code": "SANDBOX_UNAVAILABLE",
        "message": "沙箱不可用",
        "details": {
            "domainID": "tenant-001",
            "sandboxID": "sb-12345"
        }
    }
}
```

**异常类型到 HTTP 状态码映射**:
- `ValidationError` → HTTP 400
- `SandboxUnavailableError` → HTTP 503
- `SandboxTimeoutError` → HTTP 503
- `Exception` → HTTP 500
