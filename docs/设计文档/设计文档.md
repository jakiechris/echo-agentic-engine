# Echo-Agentic-Engine 设计文档

## 文档说明

本文档是 Engine 服务设计的核心主干，聚焦于三个部分：
1. **对外接口定义** - Engine 提供的所有 API 接口详细定义
2. **接口调用流程** - 每个接口触发的完整调用链和模块交互
3. **模块划分与职责** - 模块的功能边界、输入输出定义

具体模块的实现细节参见同级目录下的独立文档。

---

## 1. 对外接口定义

Engine 对外提供两类接口：
- **业务代理接口**：接收 Echo-Agent 请求，代理到沙箱内 OpenCode Serve
- **管理接口**：沙箱生命周期管理、状态查询

### 1.1 业务代理接口

#### 1.1.1 请求代理接口

**接口路径**: `ANY /trans/{path}`

**功能说明**: 将请求透传到沙箱内 OpenCode Serve，自动处理沙箱创建、健康检查、故障恢复

**请求规范**:

```http
{METHOD} /trans/{path} HTTP/1.1
Host: {engine_host}:{engine_port}
X-Domain-ID: {domainID}
X-Sandbox-ID: {sandboxID}
Content-Type: application/json
[其他 OpenCode 原生 Headers]

{OpenCode 原生请求体}
```

**路径参数**:
- `{path}`: OpenCode 原生 API 路径（如 `api/chat`, `api/files/read`）

**Headers 必需参数**:
- `X-Domain-ID` (string): 租户标识，用于存储隔离和沙箱管理
- `X-Sandbox-ID` (string): 沙箱标识，用于沙箱定位
- 其他 OpenCode 原生 Headers 完整透传

**请求体**:
- 完整透传 OpenCode 原生请求体（JSON 格式）

**响应规范**:

**成功响应** (HTTP 200):
```json
{
  "status": "success",
  "data": {OpenCode 原生响应}
}
```

**失败响应** (HTTP 4xx/5xx):
```json
{
  "status": "error",
  "error": {
    "code": "ERROR_CODE",
    "message": "错误描述",
    "details": {}
  }
}
```

**错误码定义**:

| HTTP 状态码 | 错误码 | 说明 |
|------------|--------|------|
| 400 | MISSING_DOMAIN_ID | 缺少 X-Domain-ID Header |
| 400 | MISSING_SANDBOX_ID | 缺少 X-Sandbox-ID Header |
| 503 | SANDBOX_UNAVAILABLE | 沙箱不可用（创建失败、健康检查失败） |
| 503 | SANDBOX_TIMEOUT | 请求沙箱超时 |
| 500 | INTERNAL_ERROR | Engine 内部错误 |

**行为说明**:

1. **请求验证**: 提取并验证 `X-Domain-ID`, `X-Sandbox-ID`
2. **沙箱检查**: 检查本地是否存在目标沙箱
3. **沙箱创建**: 若不存在，自动创建沙箱（准备 NAS 目录、分配端口、启动 OpenCode Serve）
4. **健康检查**: 若沙箱存在但健康检查失败，自动重建
5. **请求代理**:
   - 路径转换: `/trans/{path}` → `/{path}`
   - Headers 转换: 移除 `X-Domain-ID`, `X-Sandbox-ID`，保留其他
   - 转发到沙箱内: `http://127.0.0.1:{sandbox_port}/{path}`
6. **状态更新**: 更新沙箱活跃时间和 Redis 状态
7. **响应返回**: 完整返回 OpenCode 原生响应

**并发处理**:
- 使用 FastAPI 异步框架，每个请求独立协程处理
- 沙箱重建期间，请求在队列中排队，自动重试 3 次

### 1.2 管理接口

设计理念：所有管理接口使用 POST 方法，方法名在 URL 路径上区分，参数通过 Body 传递。

#### 1.2.1 列出所有沙箱

**接口路径**: `POST /admin/sandbox/list`

**功能说明**: 列出 Engine 管理的所有本地沙箱，包含健康状态、资源占用等信息

**请求规范**:

```http
POST /admin/sandbox/list HTTP/1.1
Content-Type: application/json

{}
```

**请求体**: 无参数（空 JSON 对象）

**响应规范**:

```json
{
  "status": "success",
  "data": {
    "total": 2,
    "sandboxes": [
      {
        "domainID": "tenant-001",
        "sandboxID": "sb-12345",
        "status": "running",
        "pid": 12345,
        "port": 30001,
        "createdAt": "2026-03-14T10:30:00Z",
        "lastActiveAt": "2026-03-14T12:45:30Z",
        "idleSeconds": 120,
        "memoryUsage": {
          "rss": 256,
          "vsz": 512,
          "unit": "MB"
        },
        "healthStatus": "healthy"
      },
      {
        "domainID": "tenant-002",
        "sandboxID": "sb-67890",
        "status": "destroying",
        "pid": null,
        "port": null,
        "createdAt": "2026-03-14T11:00:00Z",
        "lastActiveAt": "2026-03-14T13:00:00Z",
        "idleSeconds": 3600,
        "memoryUsage": null,
        "healthStatus": "unhealthy"
      }
    ]
  }
}
```

**响应字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| `total` | integer | 本地沙箱总数 |
| `sandboxes` | array | 沙箱列表 |
| `sandboxes[].domainID` | string | 租户标识 |
| `sandboxes[].sandboxID` | string | 沙箱标识 |
| `sandboxes[].status` | string | 沙箱状态：`running`, `destroying`, `error` |
| `sandboxes[].pid` | integer \| null | 进程 ID，沙箱销毁后为 null |
| `sandboxes[].port` | integer \| null | 宿主机端口，沙箱销毁后为 null |
| `sandboxes[].createdAt` | string (ISO 8601) | 创建时间 |
| `sandboxes[].lastActiveAt` | string (ISO 8601) | 最后活跃时间 |
| `sandboxes[].idleSeconds` | integer | 空闲时长（秒） |
| `sandboxes[].memoryUsage` | object \| null | 内存占用，沙箱销毁后为 null |
| `sandboxes[].healthStatus` | string | 健康状态：`healthy`, `unhealthy`, `unknown` |

**行为说明**:
- 查询本地内存中所有沙箱实例
- 补充 Redis 中的元数据（创建时间、最后活跃时间）
- 实时获取进程资源占用（通过 PID）
- 执行健康检查（发送 `GET /global/health` 到沙箱）

**功能说明**: 创建并启动沙箱（如已存在则重建）

**请求规范**:

```http
POST /admin/sandbox/create HTTP/1.1
Content-Type: application/json

{
  "domainID": "tenant-001",
```

**请求体参数**:

| 参数 | 类型 | 必需 | 说明 |

| `domainID` | string | 是 | 租户标识 |
| `sandboxID` | string | 是 | 沙箱标识 |

**响应规范**:

```json
{
  "status": "success",
  "data": {
    "domainID": "tenant-001",
    "sandboxID": "sb-12345",
    "pid": 12345,
    "port": 30001,
    "nasPath": "/nas/tenant-001/sb-12345",
    "status": "running",
    "createdAt": "2026-03-14T10:30:00Z"

```

**错误响应**:

```json
{
  "status": "error",
  "error": {
    "code": "PORT_ALLOCATION_FAILED",
    "message": "端口分配失败，无可用端口",
    "details": {}
  }
}
```

**错误码**:

| HTTP 状态码 | 错误码 | 说明 |
|------------|--------|------|
| 400 | MISSING_DOMAIN_ID | 缺少 domainID 参数 |
| 400 | MISSING_SANDBOX_ID | 缺少 sandboxID 参数 |

| 503 | NAS_PREPARATION_FAILED | NAS 目录准备失败（权限、磁盘空间） |

**行为说明**:

- 准备 NAS 目录结构
- 分配宿主机端口（30001-40000 范围内随机分配）
- 生成随机密码（UUID）
- 构建 Bubblewrap 命令
- 启动 OpenCode Serve
- 写入 Redis 沙箱元数据
- 启动空闲计时器

**请求体参数**:

```json
    "domainID": "tenant-001",

    "port": 30001,
    "portRecycled": true
  }
}
```

**错误响应**:

```json
{
  "status": "error",
  "error": {
    "code": "SANDBOX_NOT_FOUND",
    "message": "沙箱不存在",
    "details": {
      "domainID": "tenant-001",
      "sandboxID": "sb-99999"
    }
  }
}
```

**错误码**:

| HTTP 状态码 | 错误码 | 说明 |
|------------|--------|------|
| 400 | MISSING_DOMAIN_ID | 缺少 domainID 参数 |
| 400 | MISSING_SANDBOX_ID | 缺少 sandboxID 参数 |
| 404 | SANDBOX_NOT_FOUND | 沙箱不存在 |
| 500 | PROCESS_KILL_FAILED | 进程终止失败 |
| 500 | INTERNAL_ERROR | 内部错误 |

**行为说明**:
- 检查沙箱是否存在
- 标记沙箱状态为 `destroying`（写入 Redis）
- 终止沙箱进程（发送 SIGTERM）
- 等待进程退出（最多 10 秒）
- 强制终止（SIGKILL，若进程未退出）
- 回收端口到端口池
- 删除 Redis 沙箱记录
- 从本地内存移除沙箱实例

## 2. 接口调用流程与模块交互

本节描述每个接口触发的完整调用链，以及各模块之间的交互关系。

### 2.1 请求代理接口调用流程

**触发场景**: Echo-Agent 发送请求到 `ANY /trans/{path}`

**完整调用流程**:

```
请求到达 Engine
│
├─ 1. RequestValidator 模块
│   ├─ 提取 Headers: X-Domain-ID, X-Sandbox-ID
│   ├─ 验证参数存在性
│   └─ 返回 domainID, sandboxID
│
├─ 2. SandboxManager 模块
│   ├─ 查询本地沙箱: getSandbox(domainID, sandboxID)
│   │   ├─ 存在 → 返回沙箱实例
│   │   └─ 不存在 → 触发创建流程
│   │
│   ├─ [创建流程] createSandbox(domainID, sandboxID)
│   │   ├─ 2.1 PortAllocator 模块
│   │   │   └─ 分配端口: allocatePort()
│   │   │       ├─ 查询 Redis 获取已占用端口
│   │   │       ├─ 随机选择端口（30001-40000）
│   │   │       └─ 返回可用端口
│   │   │
│   │   ├─ 2.2 NasManager 模块
│   │   │   └─ 准备目录: prepareDirectory(domainID, sandboxID)
│   │   │       ├─ 创建 /nas/{domainID}/{sandboxID}/
│   │   │       ├─ 创建 defaultProject/
│   │   │       ├─ 创建 .config/.opencode/
│   │   │       └─ 返回 NAS 路径
│   │   │
│   │   ├─ 2.3 BubblewrapLauncher 模块
│   │   │   └─ 启动沙箱: launchSandbox(nasPath, port)
│   │   │       ├─ 生成随机密码
│   │   │       ├─ 构建 bwrap 命令
│   │   │       ├─ 启动进程
│   │   │       ├─ 等待就绪
│   │   │       └─ 返回 PID
│   │   │
│   │   ├─ 2.4 RedisClient 模块
│   │   │   └─ 写入元数据: setSandboxMetadata(domainID, sandboxID, metadata)
│   │   │       ├─ Key: sandbox:{domainID}:{sandboxID}
│   │   │       ├─ Value: {pid, port, status, createdAt, lastActiveAt}
│   │   │       └─ 返回成功
│   │   │
│   │   └─ 返回沙箱实例
│   │
│   ├─ [健康检查流程] checkSandboxHealth(sandbox)
│   │   ├─ HealthChecker 模块
│   │   │   └─ 发送 GET /global/health 到沙箱
│   │   │       ├─ 成功 → 返回 healthy
│   │   │       └─ 失败 → 触发重建
│   │   │
│   │   └─ [重建流程] rebuildSandbox(domainID, sandboxID)
│   │       ├─ 销毁旧沙箱: destroySandbox()
│   │       └─ 创建新沙箱: createSandbox()
│   │
│   └─ 返回健康沙箱实例
│
├─ 3. RequestProxy 模块
│   ├─ 路径转换: /trans/{path} → /{path}
│   ├─ Headers 转换: 移除 X-Domain-ID, X-Sandbox-ID
│   ├─ 构建请求: http://127.0.0.1:{port}/{path}
│   └─ 发送请求到沙箱
│
├─ 4. RedisClient 模块
│   └─ 更新活跃时间: updateLastActive(domainID, sandboxID)
│
└─ 5. 返回响应
    └─ 完整返回 OpenCode 原生响应
```

**关键模块交互**:
- **RequestValidator**: 参数验证和提取
- **SandboxManager**: 沙箱生命周期管理的核心协调者
- **PortAllocator**: 端口分配和回收
- **NasManager**: NAS 目录准备
- **BubblewrapLauncher**: Bubblewrap 进程启动
- **HealthChecker**: 沙箱健康检查
- **RedisClient**: Redis 状态同步
- **RequestProxy**: HTTP 请求代理

```
└─ 4. 返回聚合响应
    └─ 合并沙箱实例、健康状态、元数据
```

### 2.3 后台任务流程

#### 2.3.1 空闲超时检测

**触发机制**: 定时任务，每 60 秒执行一次

**调用流程**:

```
定时器触发
│
├─ 1. IdleMonitor 模块
│   └─ 扫描所有沙箱: scanIdleSandboxes()
│       ├─ 遍历本地所有沙箱
│       ├─ 计算空闲时长
│       └─ 返回超时沙箱列表
│
├─ 2. SandboxManager 模块
│   └─ 批量销毁: batchDestroy(超时沙箱)
│       └─ 对每个超时沙箱调用 destroySandbox()
│
└─ 3. 记录日志
    └─ 记录销毁的沙箱信息
```

---

#### 2.3.2 健康检查巡检

**触发机制**: 定时任务，每 30 秒执行一次

**调用流程**:

```
定时器触发
│
├─ 1. HealthChecker 模块
│   └─ 批量健康检查: checkAllHealth(sandboxes)
│       ├─ 并发发送 GET /global/health
│       └─ 返回不健康沙箱列表
│
├─ 2. SandboxManager 模块
│   └─ 批量重建: batchRebuild(不健康沙箱)
│       └─ 对每个不健康沙箱调用 rebuildSandbox()
│
└─ 3. 记录日志
    └─ 记录重建的沙箱信息
```

## 3. 模块划分与职责

本节定义 Engine 的核心模块，每个模块职责单一、边界清晰，具体实现参见独立设计文档。

### 3.1 模块总览

```
Echo-Agentic-Engine
│
├─ API Layer (接口层)
│   ├─ RequestValidator     # 请求验证与参数提取
│   └─ ResponseBuilder      # 响应构建与错误处理
│
├─ Core Layer (核心层)
│   ├─ SandboxManager       # 沙箱生命周期管理（核心协调者）
│   ├─ RequestProxy         # HTTP 请求代理
│   └─ ConfigManager        # 配置管理
│
├─ Infrastructure Layer (基础设施层)
│   ├─ PortAllocator        # 端口分配与回收
│   ├─ NasManager           # NAS 目录管理
│   ├─ BubblewrapLauncher   # Bubblewrap 进程启动
│   └─ RedisClient          # Redis 客户端
│
├─ Monitoring Layer (监控层)
│   ├─ HealthChecker        # 沙箱健康检查
│   ├─ ResourceMonitor      # 资源监控
│   ├─ IdleMonitor          # 空闲超时监控
│   └─ LogReader            # 日志读取
│
└─ Background Tasks (后台任务)
    ├─ IdleCleanupTask      # 空闲沙箱清理任务
    └─ HealthCheckTask      # 健康检查巡检任务
```

### 3.4 Infrastructure Layer (基础设施层)

#### 3.4.1 PortAllocator 模块

**职责**: 端口分配与回收管理

**输入参数**:
- `port`: 端口号

**输出参数**:
- `port`: 分配的端口号

**核心方法**:
- `allocatePort() -> int`
  - 从端口池（30001-40000）随机分配一个可用端口
  - 查询 Redis 获取已占用端口，避免冲突
  - 返回可用端口
- `recyclePort(port) -> bool`
  - 回收端口到端口池
  - 从 Redis 已占用端口集合中移除

**Redis Key**:
- `engine:ports:allocated` (Set): 已分配的端口集合

**设计文档**: [设计文档/PortAllocator.md](./PortAllocator.md)

#### 3.4.2 NasManager 模块

**职责**: NAS 目录准备与管理

**输入参数**:
- `domainID`: 租户标识
- `sandboxID`: 沙箱标识

**输出参数**:
- `nasPath`: NAS 目录路径

**核心方法**:
- `prepareDirectory(domainID, sandboxID) -> str`
  - 创建 `/nas/{domainID}/{sandboxID}/`
  - 创建 `defaultProject/`
  - 创建 `.config/.opencode/`
  - 设置权限（root 用户，读写权限）
  - 返回 NAS 路径
- `removeDirectory(domainID, sandboxID) -> bool`
  - 删除沙箱目录（可选，通常保留数据）

**设计文档**: [设计文档/NasManager.md](./NasManager.md)

---

#### 3.4.3 BubblewrapLauncher 模块

**职责**: Bubblewrap 进程启动与管理

**输入参数**:
- `nasPath`: NAS 目录路径
- `port`: 宿主机端口

**输出参数**:
- `pid`: 进程 ID

**核心方法**:
- `launchSandbox(nasPath, port) -> (pid, password)`
  - 生成随机密码（UUID）
  - 构建 Bubblewrap 命令
  - 启动进程
  - 等待进程就绪（健康检查）
  - 返回 PID 和密码
- `killSandbox(pid) -> bool`
  - 终止进程（SIGTERM）
  - 等待进程退出
  - 强制终止（SIGKILL，若未退出）
- `isProcessAlive(pid) -> bool`
  - 检查进程是否存活

**Bubblewrap 命令模板**:
```bash
bwrap \
  --unshare-pid \
  --unshare-net \
  --bind {nasPath} /sandbox \
  --ro-bind /usr /usr \
  --ro-bind /lib /lib \
  --ro-bind /lib64 /lib64 \
  --ro-bind /bin /bin \
  --dev /dev \
  --proc /proc \
  --tmpfs /tmp \
  --tmpfs /var \
  --port {hostPort}:4096 \
  --new-session \
  --die-with-parent \
  --chdir /sandbox/defaultProject \
  /usr/local/bin/opencode serve --hostname=127.0.0.1 --port=4096 --password={password}
```

**设计文档**: [设计文档/BubblewrapLauncher.md](./BubblewrapLauncher.md)

#### 3.4.4 RedisClient 模块

**职责**: Redis 客户端，沙箱元数据存储与查询

**输入参数**:
- `domainID`: 租户标识
- `sandboxID`: 沙箱标识
- `metadata`: 沙箱元数据

**输出参数**:
- `metadata`: 沙箱元数据字典

**核心方法**:
- `setSandboxMetadata(domainID, sandboxID, metadata) -> bool`
  - Key: `sandbox:{domainID}:{sandboxID}`
  - Value: `{pid, port, status, createdAt, lastActiveAt, ...}`
- `getSandboxMetadata(domainID, sandboxID) -> dict | None`
  - 查询沙箱元数据
- `deleteSandboxMetadata(domainID, sandboxID) -> bool`
  - 删除沙箱元数据
- `updateLastActive(domainID, sandboxID) -> bool`
  - 更新最后活跃时间
- `getBatchMetadata(sandboxIDs) -> dict`
  - 批量获取元数据

**Redis Key 设计**:
- `sandbox:{domainID}:{sandboxID}` (Hash): 沙箱元数据
  - `pid`: 进程 ID
  - `port`: 宿主机端口
  - `status`: 状态 (`running` | `destroying` | `error`)
  - `createdAt`: 创建时间
  - `lastActiveAt`: 最后活跃时间

**Redis 故障处理**:
- Redis 故障期间，Engine 直接拒绝所有请求
- 返回具体错误信息，说明 Redis 连接问题
- 不使用本地缓存，避免同步复杂性和数据不一致

**设计文档**: [设计文档/RedisClient.md](./RedisClient.md)

### 3.5 Monitoring Layer (监控层)

#### 3.5.1 HealthChecker 模块

**职责**: 沙箱健康检查

**输入参数**:
- `sandbox`: 沙箱实例对象

**输出参数**:
- `healthStatus`: 健康状态 (`healthy` | `unhealthy` | `unknown`)

**核心方法**:
- `checkHealth(sandbox) -> str`
  - 发送 `GET /global/health` 到 `http://127.0.0.1:{port}/global/health`
  - 超时时间：5 秒
  - 成功响应：`{"healthy": true, "version": "string"}`
  - 返回健康状态
- `checkAllHealth(sandboxes) -> dict`
  - 并发健康检查所有沙箱
  - 返回 `{sandboxID: healthStatus}` 映射

#### 3.5.2 ResourceMonitor 模块

**职责**: 沙箱资源监控

**输入参数**:
- `pid`: 进程 ID

**输出参数**:
- `ResourceUsage`: 资源占用对象

**核心方法**:
- `getResourceUsage(pid) -> ResourceUsage`
  - 读取 `/proc/{pid}/stat`

  - 返回资源占用

**ResourceUsage 结构**:
```json
{
  "cpu": {

```

#### 3.5.3 IdleMonitor 模块

**职责**: 空闲超时监控

**输入参数**:
- `sandboxes`: 沙箱列表

**输出参数**:
- `idleSandboxes`: 超时沙箱列表

**核心方法**:
- `scanIdleSandboxes() -> List[Sandbox]`

  - 空闲时长 > 配置的超时时长 → 加入超时列表

**设计文档**: [设计文档/IdleMonitor.md](./IdleMonitor.md)

## 5. 错误处理与异常定义

### 5.1 异常层次结构

```
EngineException (基类)
│
├─ ValidationError           # 参数验证错误
│   ├─ MissingDomainIDError
│   └─ MissingSandboxIDError
│
├─ SandboxException          # 沙箱相关错误
│   ├─ SandboxNotFoundError
│   ├─ SandboxUnavailableError
│   ├─ PortAllocationError
│   ├─ NasPreparationError
│   └─ BubblewrapStartError
│
├─ RedisException            # Redis 相关错误
│   ├─ RedisConnectionError
│   └─ RedisOperationError
│
└─ ProxyException            # 代理相关错误
    ├─ ProxyTimeoutError
    └─ ProxyConnectionError
```

### 5.2 异常处理策略

- **ValidationError**: 返回 400 错误，记录警告日志
- **SandboxException**: 返回 503 错误，记录错误日志
- **RedisException**: 返回 503 错误，记录严重错误日志
- **ProxyException**: 返回 502/504 错误，记录错误日志
- **未知异常**: 返回 500 错误，记录严重错误日志，堆栈跟踪

### A. 技术选型

- **Web 框架**: FastAPI（异步、高性能、自动 API 文档）
